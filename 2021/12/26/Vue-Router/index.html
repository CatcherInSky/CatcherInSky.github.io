前端路由模式

前端路由就是匹配不同的 url 路径，进行解析，然后动态的渲染出区域 html 内容。
存在的问题是 url 每次变化的时候，都会造成页面的刷新。那解决问题的思路便是在改变 url 的情况下，保证页面的不刷新。
下面是解决这个问题的方案。

hash

最常用的模式。
浏览器规定 hash 后面值的变化不会向服务器发出请求，也不会刷新页面。原本用来在作锚点在页面内进行导航。而 hash 值的变化还会触发 hashchange 这个事件。可以监听这个事件来更新页面部分内容。

history

需要后端配合的模式。
HTML5 标准发布之后新增 pushState 和 replaceState 两个 API。这两个 API 可以改变 URL 地址且不会发送请求。popstate 事件可以用来监听事件。原理和 hash 相同，没有#更美观，但是仍然会给服务器发请求。需要服务器配合，把所有路由都重定向到根页面。
